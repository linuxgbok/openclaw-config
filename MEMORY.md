## 备份与恢复

### 备份命令
```bash
~/.openclaw/scripts/backup-config.sh [备注]
```
- 备份 `openclaw.json` + 完整 `.openclaw` 目录
- 自动保留最近 20 个备份
- 创建 `latest.json` 和 `latest.tar.gz` 软链接

### 恢复命令
```bash
# 恢复配置
~/.openclaw/scripts/restore-config.sh <备份文件>

# 紧急恢复（交互式）
~/.openclaw/scripts/emergency-restore.sh
```
- 支持完整 `.tar.gz` 恢复
- 自动停止/启动 Gateway

## 用户需求处理规则

### 需求分析规则（必须遵守）
1. 不立刻给方案，先判断缺哪些关键信息
2. 按优先级只问最关键的 3-7 个问题，不要一次问太多废话
3. 主动想到可能遗漏的部分，尤其：
   - 目标是什么
   - 给谁用
   - 使用场景
   - 输入和输出是什么
   - 成功标准是什么
   - 时间限制
   - 预算限制
   - 技术限制
   - 风险和边界
   - 哪些是必须，哪些是可选
4. 需求模糊时，先整理成：目标 / 已知条件 / 缺失信息 / 建议补充
5. 信息足够后才输出正式方案
6. 不乱补设定，不确定就问
7. 输出风格：短、直接、结构化
